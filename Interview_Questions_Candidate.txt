You must design an API to update only one field (e.g., email) of a user. Which HTTP method?

Answer:

Use PATCH (partial update)
Do not use PUT unless replacing the full resource

API call succeeds but nothing is returned. Which status code?
Answer:
✅ 204 No Content
Used for:

DELETE success
Update without response body

Path Parameter

Difference between extract() and path()
extract()-Get full responsepath
path()-Extract single value

extract() is used after then() to obtain the response or a value from the response chain.
path() is used on an existing response object to read a specific json field.
extract creates or returns the response, while path reads value from it.

path-
Already have Response
Response response = requestfactory.createbooking(requestpayload)
Now extract values
int bookingid = response.path("bookingid")

extract-
Don't have response
int booingid = given().body(requestpayload).when().post("/booking").then().statuscode(200).extract().path("bookingid") or extract().response()

Question:
Verify that /users/2 returns status code 200 and the user’s id is 2.
Answer:
given()
.when()
    .get("/users/2")
.then()
    .statusCode(200)
    .body("id", equalTo(2));

Question:
Extract email from the response and print it.
Answer:
Response response =
    given()
    .when()
        .get("/users/2")
    .then()
        .extract().response();

String email = response.path("email");
System.out.println(email);


Ques- Suppose you added a new user using an HTTP POST request in Postman with the payload:
    {
      "name": "john doe"
    }
    What validations would you perform to verify that the user was added successfully?
Ans- 1- Check HTTP Status Code (Status: 201 Created)
2- Validate Response Body
Ensure the response contains expected fields:
{
  "id": 101,
  "name": "john doe",
  "message": "User created successfully"
}
3. Verify Data via GET API
Make a GET request to confirm user was actually saved:
GET /api/users/101
Response should:
Return the newly created user
Match same data
4. Validate Headers
Check important headers:
✅ Content-Type: application/json
✅ Location header (best practice)
Location: /api/users/101
5. Database Validation (if accessible)
✅ Check DB directly:
SELECT * FROM users WHERE name = 'john doe';
